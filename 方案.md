# 硬盘状态监听器 - 开发方案

## 一、范围与目标

### 1.1 产品范围
- 目标平台：Windows、Linux
- 开发环境：可在 WSL 中开发与调试，但不将 WSL 作为正式兼容目标
- 界面形态：命令行界面（CLI）

### 1.2 核心目标
软件启动后自动扫描磁盘（Windows 逻辑分区 / Linux 挂载点），在发生读写时输出：
- 磁盘标识
- 状态（READ / WRITE）
- PID
- 进程名
- 实时速率（KB/s）

## 二、技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| 编程语言 | Python 3.10+ | 生态成熟、开发效率高 |
| Windows 事件采集 | ETW | 可获取进程级文件 I/O 事件，支持精确归因 |
| Linux 事件采集 | eBPF/tracepoint | 可在内核侧捕获 I/O 事件并关联 PID |
| 进程信息补充 | psutil | 进程名/状态补充与兜底能力 |
| CLI 渲染 | 标准输出 + 清屏重绘 | 实现简单、可控性高 |

### 核心依赖（建议）
- `psutil>=5.9.0`
- `typing-extensions>=4.0.0`（如需兼容较低 Python 版本的类型特性）
- Windows ETW 与 Linux eBPF 相关依赖在各平台单独安装（按实现阶段确定具体库）

### 数据流
1. `collector` 持续采集原始 I/O 事件
2. `aggregator` 按时间窗聚合 `(disk, pid, op)`
3. `display` 定时读取聚合结果并清屏重绘

## 四、核心实现方案

### 4.1 Windows：基于 ETW 的精确归因
- 采集字段：`timestamp, pid, operation(read/write), bytes, file_path`
- 通过 `file_path` 映射盘符（如 `C:`、`D:`）
- 聚合维度：`(disk, pid, operation)`
- 速率计算：`rate_kb_s = bytes_sum / window_seconds / 1024`

说明：该方案解决“只能拿到进程总 I/O、不能按磁盘归因”的问题。

### 4.2 Linux：基于 eBPF/tracepoint 的事件采集
- 采集字段：`timestamp, pid, rw, bytes, dev(major:minor)`
- 通过 `/proc/self/mountinfo` 建立 `dev -> mountpoint` 映射
- 聚合维度同 Windows，统一输出模型

### 4.3 聚合与状态判定
- 聚合窗口默认 `1s`（可配置）
- 每个窗口内：
  - `read_bytes > 0` 输出 `READ`
  - `write_bytes > 0` 输出 `WRITE`
  - 读写同时存在可分两行显示，或以主导方向显示（默认分两行）
- 输出仅保留活跃项（窗口内 bytes > 0）

### 4.4 自适应刷新 + 清屏重绘
按你的要求，采用清屏重绘，不使用局部差量渲染。

- 清屏方式：`cls`（Windows）/`clear`（Linux）
- 自适应刷新策略：
  - 空闲期（无活跃 I/O）：每 `1.5s` 重绘
  - 活跃期（有活跃 I/O）：每 `0.3s` 重绘
- 轮询与渲染解耦：采集连续进行，渲染按刷新节奏触发

### 4.5 性能控制（问题4的落地措施）
- 事件驱动替代“高频全进程轮询”
- 进程名缓存：`pid -> process_name`，TTL 默认 30 秒
- 有界队列：防止突发 I/O 造成内存无限增长
- 背压策略：当队列积压超过阈值时，优先保留大块 I/O 事件
- 输出限流：每个磁盘最多显示 Top N 活跃进程（默认 10）

### 4.6 降级模式（可选但建议）
当 ETW/eBPF 不可用（权限或环境限制）时：
- 启用 `psutil` 近似模式
- 明确标注：`[Approx Mode] 进程与磁盘归因可能不精确`

## 五、命令行参数

```bash
python main.py \
  --window-ms 1000 \
  --active-refresh-ms 300 \
  --idle-refresh-ms 1500 \
  --top-n 10
```

参数说明：
- `--window-ms`：聚合窗口大小（默认 1000）
- `--active-refresh-ms`：活跃期刷新间隔（默认 300）
- `--idle-refresh-ms`：空闲期刷新间隔（默认 1500）
- `--top-n`：每磁盘展示的最多进程数（默认 10）

## 七、验收标准

1. 能扫描并显示所有目标磁盘（Windows 逻辑分区 / Linux 挂载点）
2. 发生读写时，能输出磁盘、PID、进程名、READ/WRITE、速率
3. 速率计算正确，单位为 KB/s
4. CLI 可持续刷新，无明显卡顿
5. 在活跃负载下进程 CPU 占用可控（目标 < 15%，以 1 秒窗口、Top10 为基线）
6. 明确支持 Windows 与 Linux；WSL 仅作为开发环境

## 八、GUI扩展接口

采用回调模式，将 `display.py` 作为默认CLI实现，可替换为GUI：

```python
# ui/callback.py
from abc import ABC, abstractmethod
from typing import List
from core.models import DiskActivity

class OutputCallback(ABC):
    @abstractmethod
    def output(self, activities: List[DiskActivity], interval_ms: int) -> None:
        pass

    @abstractmethod
    def set_refresh_interval(self, interval_ms: int) -> None:
        pass
```

使用方式：
```python
# CLI模式（默认）
cli_callback = CLIDisplayCallback()
monitor = DiskMonitor(callback=cli_callback)

# GUI模式（扩展）
gui_callback = MyGUIWidget()
monitor = DiskMonitor(callback=gui_callback)
```
